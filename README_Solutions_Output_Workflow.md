# Notebook solution/student workflow

This package uses one editable instructor-solutions notebook per lab. Red solution cells are tagged with `solution` and `solution-red`. The student notebooks are generated by removing those tagged cells and clearing every code-cell output.

## Main files

- `Dispersion_Chromaticity_Local_Lab_Instructor_Solutions.ipynb` — instructor version with red answers and executed outputs.
- `Dispersion_Chromaticity_Local_Lab_Student.ipynb` — generated student version, no answers and no code outputs.
- `Quadrupole_Focusing_Xsuite_Instructor_Solutions.ipynb` — instructor version with red answers and executed outputs.
- `Quadrupole_Focusing_Xsuite_Student.ipynb` — generated student version, no answers and no code outputs.

## Scripts

Run from the directory containing the notebooks:

```bash
python3 make_student_versions.py
python3 check_student_versions.py
```

`make_student_versions.py` removes cells tagged `solution`, `solution-code`, `solution-red`, `instructor-note`, or `scratch`, then clears outputs and execution counts in the remaining student-facing cells.

`check_student_versions.py` checks for leaked solution cells, leaked workflow tags, widget-state metadata, and uncleared outputs.

## Execution note

The instructor notebooks were executed with ordinary calculation and plotting cells included. Interactive slider cells are tagged `skip-execution` for the packaging execution step because pre-executing widget callbacks can hang or inflate the notebook. Those cells include a visible placeholder output; run them inside Jupyter to activate the sliders.
